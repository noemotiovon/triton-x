# Skill: Convert CUDA Triton Kernel to NPU Triton Kernel

## Description

This skill defines the rules, patterns, and process for converting a CUDA Triton Kernel into an NPU (Ascend) Triton Kernel. It is the core skill of the Triton-X project and should be continuously updated with new patterns and lessons learned.

## When to Use

When the user asks to convert a kernel, such as:
- "把这个 CUDA kernel 转换成 NPU 版本"
- "Convert fla's fused_recurrent to Ascend"
- "生成 NPU 的 Triton Kernel"

## Conversion Process

### Step 1: Analyze the Source Kernel

Before converting, analyze the CUDA Triton kernel to understand:
1. **Input/Output tensors**: shapes, dtypes, memory layout
2. **Parallelism model**: how `program_id` / grid is used
3. **Memory operations**: `tl.load`, `tl.store`, pointer arithmetic
4. **Computation patterns**: reductions, element-wise ops, fused operations
5. **Special features**: atomics, shared memory, synchronization

### Step 2: Apply Conversion Rules

Apply the rules from the **Conversion Rule Table** below. For each pattern in the CUDA kernel, find the corresponding NPU pattern.

### Step 3: Generate NPU Kernel

1. Create the NPU kernel file in the appropriate directory:
   - Upstream library → `src/{lib}/ascend/generated/{kernel_name}.py`
   - Native kernel → `src/native/ascend/{kernel_name}.py`
2. Add the file header:
   ```python
   # Generated by Triton-X
   # Source: src/{lib}/cuda/{kernel_name}.py
   # Date: YYYY-MM-DD
   ```

### Step 4: Update Manifest

Update `manifest.yaml` with:
- `ascend_path`: path to the generated file
- `source`: `"generated"`
- `status`: `"converted"`
- `last_updated`: today's date

### Step 5: Proceed to Testing

After conversion, check for accuracy tests:
- If test exists → run it (see `generate-accuracy-test` skill)
- If no test → generate one first

### Pattern: Element-wise Operation

**CUDA:**
```python
@triton.jit
def add_kernel(x_ptr, y_ptr, out_ptr, n, BLOCK: tl.constexpr):
    pid = tl.program_id(0)
    offsets = pid * BLOCK + tl.arange(0, BLOCK)
    mask = offsets < n
    x = tl.load(x_ptr + offsets, mask=mask)
    y = tl.load(y_ptr + offsets, mask=mask)
    tl.store(out_ptr + offsets, x + y, mask=mask)
```
