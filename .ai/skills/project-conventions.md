## Naming Conventions

- **Kernel files**: `{kernel_name}.py` вЂ” snake_case, descriptive of the operation
- **Test files**: `test_{kernel_name}.py` вЂ” one test file per kernel
- **Manifest files**: `manifest.yaml` вЂ” one per library directory under `src/`

## Manifest Management

Every time a kernel is added, converted, or tested, update the corresponding `manifest.yaml`:

```yaml
kernels:
  - name: "kernel_name"
    cuda_path: "cuda/kernel_name.py"
    ascend_path: "ascend/generated/kernel_name.py"
    test_path: "tests/{lib}/test_kernel_name.py"
    source: "generated"       # upstream | generated | manual
    status: "pending"         # pending | converted | testing | verified | failed
    last_updated: "YYYY-MM-DD"
    notes: ""
```

**Status transitions**:
- `pending` в†’ `converted` (after NPU kernel is generated)
- `converted` в†’ `testing` (during test execution)
- `testing` в†’ `verified` (all accuracy tests passed)
- `testing` в†’ `failed` (tests failed, needs investigation)
- `failed` в†’ `converted` (after fixing and regenerating)

## File Headers

All generated NPU kernel files MUST include a header comment:

```python
# Generated by Triton-X
# Source: src/{lib}/cuda/{kernel_name}.py
# Date: YYYY-MM-DD
```

All upstream-sourced files MUST include:

```python
# Source: {upstream_repo_url}
# Commit: {commit_hash}
# Date: YYYY-MM-DD
```

## Accuracy Standards

Always read `config/accuracy.yaml` for tolerance thresholds. Use the shared `check_accuracy` function from `tests/conftest.py` for consistency.

## Key Rules

1. **Never modify files in `cuda/` or `ascend/upstream/`** вЂ” these are read-only copies from upstream
2. **Always update `manifest.yaml`** after any kernel operation
3. **Always update `config/upstream.yaml`** when integrating a new upstream library
4. **One kernel per file** вЂ” keep kernels isolated for clarity
5. **Tests must be self-contained** вЂ” each test file should import only its target kernel and standard dependencies
